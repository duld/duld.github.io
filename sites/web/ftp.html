File Transfer Protocol

FTP

FTP has been the work horse of transfering files in a client server network.
Originally written Abhay Bhushan in 1971, FTP remains an important part of the modern web.

FTP
File Transfer Protocol, used for transfering files between a remote server to a local computer. An FTP address is similar to a URL address but instead of www prefix we use the ftp prefix. So ftp.<domain name>.<top level domain>. Sometimes a port number is listed at the end of an ftp address or a username is specified.

<username>@ftp.example.com:<port number>

Some benefits of using FTP is the ability to transfer multiple directories, at one time (uploading and downloading). Overall FTP is faster than Email or HTTP when it comes to transferring files. However some pitfalls of FTP is that it is not secure by default. SFTP leverages the SSH protocol to authenticate and establish secure connections. There is a great breakdown over at digitalocean.com  on the process, which goes in depth. More than I can take in, during one sitting. Page bookmarked!

It is popular to use visual clients to interface with an FTP server. Browsers can inteface but they are not as fast and considered less reliable. Just as there are many different software choices to interface with an FTP server, there are also many different choices of software to run an FTP server. If you don't feel like installing software just to interface with an FTP server you can also use the command line to connect to an FTP server, download files or upload files.

Greater detail at wikihow
ftp ftp.someftpserver.com      (Connect to FTP server)
binary                                    (Change to binary to download non text files)
get somefilename.mp4          (download a file)
put yourownfile.giv                (upload a file from your local machine to the ftp server)
close                                     (close the connection with the FTP client)

Some server software include: FileZilla, WinSCP, Syncplify, CrushFTP and many more. When running your own FTP server you can create different user accounts with varying levels of permissions. You can control which users can read/write/delete to the FTP server.
