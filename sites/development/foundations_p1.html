Introduction to the Web

  Understanding How the web works

  • Internet vs the Web - used interchangeably but they are not the same.
  • Internet - used for
  • Email
  • SMS messaging
  • Application data
  • World Wide Web
  • World Wide Web
  • Runs over the Internet
  • Transfers data using HTTP
  • helps facilitate client to server services
  • Clients:  are applications or software that make requests to servers
  • Servers: respond to requests by clients and store or send data back

  Working with clients and servers

  • Client browser request to server lifecycle
  • browser makes a request based on a URL - Uniform Resource Locator
  • The URL request is passed to a DNS server (Domain Name Server)
  • The DNS then translates the URL into an IP address
  • The Browser then uses the IP address to locate the host server and send a request for content
  • If the webpage is not a static site, then an application on the server handles the request

  The DNS and why it matters

  • A website has it’s own unique IP address - useful for computers, complicated for humans
  • A unique domain name can map onto an IP, making it easier for humans to remember
  • the DNS lookup process:
  • your browser makes a request to retrieve a webpage
  • the browser sends a DNS lookup request to your local ISP - if the ISP knows the IP address associated with that DNS then it returns the result, otherwise
  • the local ISP sends the request a Root Server to retrieve the Name servers for the Top Level Domain (TLD)
  • ISP sends request to TLD name servers to get authoritative name servers for request URL
  • ISP then sends the request to the authoritative name servers and returns the result to you.
  • A visual explanation of how DNS lookups work
  • If you don’t have a DNS associated with your website, no one will be able to find you
  • ensure you set up the correct ownership for a site, if it’s for a client the client should be the owner of the site.

  Internet Protocols

  • In order for computers to communicate they must agree upon a protocol.
  • Each protocol is different and has it’s own advantages/disadvantages
  • Common protocols include:
  • Application Layer: HTTP, FTP, SMTP, DNS, RIP, SNMP
  • Link Layer: Wireless LAN, Ethernet, ATM
  • Transport Layer: TCP, UDP
  • Internet Layer: IPv4, IPv6

  Explore HTTP

  • Standard protocol for transferring data over the Web
  • The browser is an HTTP Client, that sends and receives content
  • FTP: File Transfer Protocol, enables transfer of large files
  • SMTP: Simple Mail Transfer Protocol, used to send email
  • POP: Post Office Protocol, used to receive email
  • RTP: Real-Time Transfer Protocol, standard for audio and video
  • HTTP is stateless, once requests are sent and received it is forgotten, due to the stateless nature of HTTP, state must recorded in Cookies, JavaScript variables or Server Session management.
  • HTTPS: Hypertext Transfer Protocol Secure, a more secure version of HTTP

  Anatomy of a URL

  • http://www.lynda.com 
  •      http:             www             lynda               com
  • <protocol> <subdomain> <domain> <top-level domain>
  • Server administrators can  create as many subdomains on the server as needed
  • default.html or index.html are the default pages a server will load unless otherwise specified. Port 80 is the default port for HTTP
  • forward slashes along with names of directories are used to navigate to different files on the server. http://www.lynda.com/search  will navigate to the search directory and load whatever the default file inside that directory.
  • when a ‘?’ is visible in a url followed by a “name=value” pattern, we are passing arguments to the server. This can cause the page to load different results based on the name value pairs.

  How browsers work

  • Typical Browser Components
  • Browser UI: Visual chrome, Address bar, other visual elements
  • Internal Engine:  Directs communication across the various components.
  • Rendering Engine: Parse HTML and CSS and renders the content on the screen
  • Networking: HTTP and other protocol components
  • Data Storage: Internal data storage and session management capabilities
  • JavaScript Interpreter: Internal JavaScript engine, executes any client-side scripts.
  • Rendering Engines:
  • Trident: Internet Explorer, Windows Mobile, AOL Browsers
  • EdgeHTML: Microsoft Edge - a fork of Trident
  • Gecko: Firefox (in transition to Servo)
  • Presto: Opera
  • WebKit: Safari, webOS, Android
  • Blink: Chrome, Opera, Android

  Web standards and the W3C

  • The Browser Wars - Netscape Navigator VS. Internet Explorer
  • 1998: web standards project:
  • Group of professionals and companies that actively supported the W3C and the adoption of web standards
  • W3C began in 1994, and issued recommendations for web tech
  • began to refer recommendations as standards
  • encouraged the use of web standards
  • www.webstandards.org
  • Importance of Standards
  • Writing to standards ensures consistency across supporting browsers
  • Web standards continue to evolve

  Web server basics

  • Web Server - A computer that processes requests via HTTP
  • All web servers have an IP address, that is registered on a DNS Server.
  • Many servers use virtual hosting, which allows multiple websites to be served from the same IP address. This is called Shared Hosting.
  • Shared hosting is often cheaper than Dedicated hosting.
  • Sites are stored on Web Servers as a series of directories and files.
  • Web Stack - a group of software that work in conjunction with each other to build and process web sites. Make sure your hosting company supports the your stack.
  • Web Stack usually consist of:
  • Operating System
  • Web Server
  • Database Server
  • Programming Language
  • LAMP - Linux (OS), Apache (server), MySQL (Database), PHP (programming language)
  • WAMP - Windows, Apache, MySQL, PHP
  • WISA - Windows, IIS, MS SQL, .NET
  • MARS - mySQL, Apache, Ruby, Solaris

Front-end Technology

  Front-end design

  • the visual layer of websites and applications, typically the UI, layout, typography, images and other visual elements and their styling
  • can refer to the usability and UX design as well
  • the core of front-end is: HTML, CSS, JavaScript
  • Recently more development incorporates client side processing inside the browser using JavaScript and powerful JS Libraries.
  • A front end developer may not have the same responsibilities as a front end designer. But their tasks could overlap depending on where they work.
  • Front end Designer responsibilities:
  • Creating mockups & visual standards
  • structuring semantic content
  • handling site accessibility
  • site typography
  • page layout
  • form design
  • interactivity and animation
  • site imagery

  HTML, the language of the web

  • The standard language used for structuring content on the web
  • HTML timeline:
  • 1995: HTML 2.0 spec released
  • 1997: HTML 3+ released, browsers continued to release proprietary features
  • 1999: HTML 4 is published as a standard. Browsers begin to adopt web standards
  • 2000: XHTML 1.0 released. To replace HTML
  • 2004: WHATWG forms to continue evolution of HTML.
  • 2008: HTML5 is adopted by W3C and given a working charter
  • 2009: XHTML 2.0 is abandoned, in favor of HTML5
  • HTML5 is the defacto standard

  Structuring HTML

  • Learning HTML boils down to
  • Learning the syntax
  • which elements are supported
  • basic structure of HTML documents
  • Every HTML page has the same basic structure: doctype, html, head, body, header and footer
  • HTML is frequently used to make sandwiches

  Controlling presentation through CSS

  • Stylesheet language designed to control the presentation of structured documents such as HTML.
  • HTML is a structural semantic language, It’s used to define the structure and content of a webpage
  • older presentation tags: <font> <b> <i> are either deprecated or redefined within the language to reduce the presentational nature of elements
  • CSS is normally stored in an external .css file, that can be linked to possibly many html files.
  • The Cascade refers to how CSS style rules apply styles from a css file, then look at styles included in the html page inside of a <style> element, then finally look at an element’s inline style.
  • CSS is made up two main parts: The Selector and the Declaration
  • inside the Declaration styles are set by specifying a CSS property name and a value, separated by a colon.

  Client-side scripting with JavaScript

  • Modern web design makes heavy use of JavaScript.
  • Allows designers to create interactive elements, can update, sort and present data directly in the browser, no server interaction required.
  • Using JavaScript you can leverage multiple Libraries to add functionality to websites.
  • experiment with plain JavaScript first before using popular Libraries or Frameworks
  • Have a strategy for providing fallback content for when JavaScript is not enabled. Why wouldn’t you have JS enabled? Because often you can’t trust many sites to be conservative in their use of JS. And you can always enable it on websites you trust, to get the desired functionality.
