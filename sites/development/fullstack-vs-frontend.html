Full Stack vs Front End

Jack of All Trades vs Specialist

Full Stack developers have knowledge of the complete build process of web page and can assist
with any aspect of it's creation. On the other hand Front End Developers
are laser focused on a specific technology stack and bring
their own unique experience to thet able.

Becoming a Web Developer: Full Stack vs Front End (lynda.com) lecture

Getting Started

  Types of developers

  • Front-end developers - HTML , CSS  and JavaScript
  • Back-end development - works on backend servers
  • Full-Stack developer - able to work both Front-end and Back-end

  The role of HTML

  • required by all web developers, even if they don’t work in it day in, day out.
  • Hyper Text Markup Language - content and structure of a webpage
  • Information Architecture (IA), describes how to best structure the design of a website’s navigation. How a user will navigate and what other content will be available to them
  • Markup Language, defines content and structure of a webpage. Some content is plain text, some content are images, video, or  links to other websites.
  • Structure and Semantics are important when working with HTML.
  • DOM - Document Object Model

The Role of CSS

  • Design of a website can be broken down into either User Experience design or User interface design: (UX) and (UI)
  • UX - Look and Feel
  • UI - Functionality of a website
  • Cascading Style Sheets (CSS) - made up of two parts
  • Rules about how the DOM is translated visually
  • Cascading Style Rules - describe the priority of the set of rules that apply the styles to the page
  • Motion can also be described using CSS and applied to DOM elements
  • Web content must be responsive and respond to user devices.
  • There have been three major CSS releases, with newer features of CSS being released in separate modules going forward.

The Role of JavaScript

  • Responsible for front end interaction
  • Browsers do not always have access to the same features of modern JavaScript
  • Newer version of JS can be transpiled into an older version of JS. (ES6->ES5)
  • NodeJS - a JS engine based on Google’s V8 and Microsoft’s Chakra
  • WebPack - a module bundler, takes modules w/dependencies and emits static assets representing those modules. Dependencies can be written in CommonJS (excerpt from: https://webpack.github.io/  )
  • GulpJS - A task runner for javascript. Used to solve build problems. Very involved topic ( stackoverflow - Task Runners )

Back end essentials

  • used to use data from a database to construct user experiences (web pages) specific to a user. This can be based on interest (youtube), shopping suggestions (amazon), account details (banking) etc.
  • A web stack is almost always unique to each company. And developers should understand the tech inside that stack as it relates to their company. Not all databases are the same
  • PHP + MySQL is currently the most popular in this regard (Java?)
  • Other options: Node.js, .NET, Rails
  • Lots of options to choose from and there are always new updates being released for back end tech
  • Which back end language should I learn in 2017 - quora

Servers

  • If you want a web page/web app you’re dealing with a server (computer).
  • server’s ‘serve’ up files to a user to render a webpage.
  • Larger sites use systems that construct the web pages on schedule
  • Front end developers build chunks of the this content or all of it on smaller pages
  • Two Main types of Servers:
  • Linux
  • cheaper
  • fully open source
  • more common
  • even windows cloud infrastructure (Azure)  has Linux options
  • Windows
  • popular with large companies
  • great dev environments
  • increased movement towards open source and transparency
  • You can practice these backend concepts by using NodeJS and ExpressJS or PHP and MYSQL

Server-side scripting

  • The language that back end developers use
  • All back end development usually solve the same tasks
  • choices include:
  • .NET
  • Python
  • NodeJS
  • ASP.NET
  • Ruby
  • PHP
  • (Java)? it’s ok Java, I recognise your existence.
  • some choices are platform specific, Ruby loves linux, Python is warming up to Microsoft and .NET’s little brother .NET core wants to be everyone’s friend.

Database languages

  • To access data if you need to use a database’s language to query data
  • Relational DataBase Management System (RBDMS) - creates tables and makes connections from fields within the tables to fields in other tables or other tables themselves. This makes up the Relational part of RBDMS
  • popular implementations of RBDMS: MS SQL , MySQL , Oracle , PostGreSQL
  • NoSQL - provides a mechanism for storage and retrieval of data that is modeled in a means other than tabular relations used in relational databases
  • common NoSQL DBs: MongoDB , CouchBase , CouchDB
  • Other database styles: column stores, key value stores

Learning and Training

Train for the industry

  • ways to train: instructor-led training, projects, self study and working with clients
  • instructor-led: school, bootcamps
  • most developers describe themselves as self-taught.
  • Majority of all learning only comes when you build something and when working with others
  • You won’t finish learning

Traditional education

  • do you need a college education?
  • some companies require a degree: estimated 65%
  • less than half developers have a degree in CS but what about self taught developers who have a degree in something else I wonder? =).
  • excellent for establishing a wide foundation to build on.

Coding bootcamps

  • hyperfocused short term training
  • 40+ hours per week, high placement (short term contract work that could be over in a month or two count as ‘placement’, so food for thought).
  • many bootcamp graduates already have a bachelor's

Other learning options

  • Youtube, tutorials, online articles - often old or outdated
  • online courses: Udemy, EdX, Udacity, Coursera, many more...
  • online communities: slack.com, discord (popular with reddit), stackoverflow
  • using publicly visible code can help show how others have solved problems you may run into. -  Github, Codepen, BitBucket
  • talk to people

Social learning opportunities

  • spending time with other developers is very important
  • most devs find jobs through their friends
  • meetup.com is a good way to find local groups
  • conferences give access to a different level of developers
  • hackathons - work with a team of developers to build a project in a competitive atmosphere

Core developer tools

  • know your editor or IDE
  • IDE - Visual Studio, Eclipse, Android Studio, NetBeans
  • Text Editor - Sublime, VIM, Atom, Visual Studio Code
  • The Terminal/Command Line - text based commands
  • The browser - view your work and debug your HTML, CSS and JS using the built in developer tools

Other essential tools

  • version control - git
  • build tools - GulpJS, Browserify
  • Libraries - jQuery, ReactJS (some argue)
  • Frameworks - Bootstrap, AngularJS, ReactJS
  • Wordpress - there are a lot of wordpress sites. a lot, like a lot.

Core concepts

  • Accessibility is important to implement into your website. It is often forgotten and depending on the site is a requirement.
  • Agile workflows: a workflow strategy that operates in short two week sprints
  • Test-Driven Development (TDD) - write tests to validate your code against

Business and leadership skills

  • communication is required
  • raise issues w/o being confrontational, listen
  • you should be able present and defend your ideas - pros, cons, think your solution through
  • leverage social media if able
  • negotiating your salary is a common problem for developers

Getting a Job

Types of jobs

  • Salary and work life culture are important when choosing a job
  • remote workers are often more satisfied by their job. Requires the ability to focus and perform w/o being distracted

Maximize compensation

  • development pays pretty well out of the gate
  • where you live and what skills you have will determine your salary potential
  • employees at larger companies earn more on average than self-employed individuals
  • The longer you stay in the industry, and increase your skills the higher your salary
  • pay by Degrees:
  • PHD: $122,000
  • Masters: 119,000
  • Bachelors: $107,000
  • Bootcamps: $112,000
  • Without passion, you will burn out.Even if you like development, where you work or the type of work that you are doing can burn you out. Poor work life balance could kill your relationships, or kill your opportunity to develop new ones.

Land a job

  • getting a referral is the best way to get a job
  • recruiters are very useful
  • scout out company employees that go to meetups or events near you
