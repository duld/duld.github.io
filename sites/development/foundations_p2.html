Front-End Technology (continued)

  Common image types

  • All image formats are not created equal. When are where to use a particular image format depends on use case.
  • JPEG: Joint Photographic Experts Group, lossy format that can have lose information (artifacts) over time. Use another lossless image to work off of and then save to jpeg. RGB
  • GIF: Graphics Interchange Format (pronounced Gh-iff, not JiPH. It is very important). 8 bit graphics limited to 256 colors. Lossless image but limited color range. Good for icons. RGBA
  • PNG: Portable Network Graphics, Lossless, supports millions of colors, larger file size. RGBA
  • SVG: Scalable Vector Graphics, XML-based vector image format with support for interactivity and animation. A little larger but remain sharp.

  What is an API?

  • API: Application Programming Interface
  • usually written to make it easier for devs to use an application or to make services publicly available.
  • Many websites will incorporate 3rd party APIs as a main feature draw. Larger sites will allow this because it increases their brand.
  • Public facing APIs will limit the kinds of data you have access to, don’t expect everything because you ask for it.
  • Be wary of depending on API’s that may disappear. There goes your feature.

  HTML5 APIs

  • HTML5 Taxonomy (as of 10/14)
  • As the web progressed HTML’s limitations on what content it could markup became noticeable. This lead to plugins such as Adobe Flash, Java applets, MS Silverlight, etc
  • HTML5 arose from the desire to expand the capabilities of HTML and reduce reliance on 3rd party plugins.
  • WHATWG - Web Hypertext Application Technology Working Group. The group responsible for the initial draft of HTML5.
  • Common HTML5 APIs
  • Media API - Integrated API designed to control the loading and playback of video and audio.
  • Drag and Drop API - enables drag and drop functionality on page elements, includes methods for handling drag and drop elements from outside the browser.
  • APP Cache API - contains mechanisms for storing data offline
  • Geolocation - access the current location of users
  • Canvas API - Standalone API associated with HTML5. Give the ability to create a canvas area within the client that can be drawn or painted on. used for games, animations or other visual effects.

  Web fonts

  • What font a user could load used to be limited to what fonts were installed on the user’s PC.
  • Aria, Verdana and Georgia; used to dominate the displayable fonts on a website.
  • Web Fonts
  • technology that allows for the downloading and temporary installation of fonts within the browser.
  • Most frequently is used to describe the CSS @font-face technique
  • Early Web Fonts Issues
  • Most licenses included provisions which would have made @font-face violate their licenses.
  • Font foundries and type designers didn’t like giving users the ability to download fonts. Fonts sell for a lot of money. Font Pack Pro goes for $800! Helvetica for a webpage with 25,000,000 views will cost $15,000. For Boring Helvetica. Who feels like getting fleeced?
  • Most font formats weren’t designed for on-screen usage.
  • Font formats Designed for web Usage
  • Embedded OpenType (EOT)
  • Web Open Font Format (WOFF)
  • designed to compress fonts for online usage.

Back-End Technology

  Server-side scripting

  • MDN Server-side first steps
  • Any scripting or programming that runs on a server is considered Server-side scripting
  • Common Server-side scripting tasks:
  • processing a form
  • managing an online purchase
  • registering a new user
  • CGI - Common Gateway Interface. Designed for languages like C++ or Perl
  • Newer languages:
  • PHP - Personal Home Page language - great for making web pages you don’t plan to share.
  • Java - A strongly typed OOP coffee, frequently found near AT&T stores.
  • .NET - Microsoft software framework, useful for catching fish.
  • CFML - Cold Fusion Markup Language - brr
  • Server-side scripting workflow
  • A web page containing server-side scripting is requested from a server
  • the web server, processes the script before serving the page is served to the browser, this could involve:
  • pulling information from a database
  • making simple calculations
  • choosing which piece of content to display on the page.
  • These requests could have multiple sources:
  • submitting a form on the page
  • interacting with a widget  The importance of PHP
  • The scripts run on the server out of view of the client, the result is sent to the client. Adding a layer of security.
  • The downside of server-side scripting
  • processing is done on the server, which can increase server load
  • processing data usually requires a page refresh on the browser
  • client must wait for server to complete processing.
  • Asynchronous Javascript And XML (AJAX) and other client-side bridges help enhance the usability of server-side scripting.
  • incorporating client side scripting along with server side scripting has become very popular. The client often has a multi-core cpu that’s just sitting around. Put that slacker to work!

  The Importance of PHP

  • PHP development Timeline
  • 1994: Rasmus Lerdof writes a set of CGI scripts to track visitors to his online resume. He names the scripts Personal Home Page
  • 1997: Andi Gutmans, Zeev Suraski & Lerdof rewrote the existing parser and released PHP 3.0.
  • Popular aspects of PHP
  • PHP was free and open source
  • large community of developers
  • paired with other open source tools:
  • Linux (OS)
  • Apache (server)
  • mySQL (database)
  • PHP (server-side language)
  • One of the most, if not the most widely used language on the web.
  • How PHP works:
  • a file is written with a .php file extension. The web server will process the file
  • As the web server processes the file it does additional language specific processing when it encounters identifiers.
  • example of an identifier: <?php echo date(“D M d, Y G:i a”); ?>
  • <?php ?> php tag, all text contained must follow proper php syntax.
  • the php code is processed on the server and strps the <?php ?> tag, from the server response.
  • PHP must be run on a server, to run php you need software to create a local web server or you need access to a server online to run your scripts.
  • Using PHP:
  • PHP is widely supported
  • there is a large pool of existing developers to work with
  • PHP is relatively easy to learn with a lot of available resources

  Dealing with data

  • Many sites require and make use of user data
  • Data can be stored locally on the client or on an external database
  • small amounts of user data can be stored as cookies on a client’s machine.
  • WebStorage API
  • allows browsers to store Key/Value pair
  • XML and JSON can be used to store larger sets of data.
  • Databases are best used suited for storing large sets of data.
  • Database can be thought of as spreadsheet tables - with connected references
  • IndexedDB: low-level API for client-side storage of significant amounts of structured data, including files/blobs.
  • Web SQL:  is a web page API for storing data in a database that can be queried using a variant of SQL.
  • Relational Databases:
  • store data in tables that relate to each other based on item key values
  • this allows for complex sorting and filtering of data
  • Oracle, mySQL, Sybase and other SQL-based databases are popular relational databases.
  • Non-Relational Databases
  • Alternative to relational databases, growing in popularity
  • NoSQL databases store data into individual flat, object-driven datasets
  • These are often faster to index, and scale easier than relational databases
  • MongoDB, CouchDB, SimpleDB, HBase, CouchBase, and Cassandra are popular NoSQL databases
  • Relational databases operate off of a Schema which can be difficult to modify later on. NoSQL databases do not have this particular problem

  Using SQL databases

  • SQL: structured query language
  • Web Apps often use SQL to update, create delete or retrieve information from a relational database (CRUD apps)
  • SQL syntax:
  • sql syntax is simple and logical, the basics are easy to learn
  • select, insert, update, delete and other simple statements are used to retrieve and manipulate data.
  • SQL
  • SELECT * FROM tours
    WHERE price < 200
    ORDER BY tourName ASC
  • ENGLISH:
  • return all items from tours table that are below $200; sort the returned values by the tour’s name in ascending order.
  • data returned from a SQL query is known as a dataset
  • Learning SQL
  • there are multiple versions of SQL, each with it’s own extensions
  • SQL queries are usually written in other web applicatio code
  • Dedicated applications like mySQL can help you create and manage databases.
  • understanding the basics of SQL is important.

  Content management systems

  • Content Management System (CMS):
  • an application designed to control the creation, management, publishing, and archiving of a site’s content
  • can allow different groups of users to create and control content for the site
  • Allows you to turn over site management over to less technical individuals
  • plugins can be added to the CMS to increase features
  • Page creation using a CMS involves:
  • selecting a template
  • adding content to the template using  WYSIWYG editor
  • publish content.
  • Exploring CMS options
  • Some CMS are hosted online and only require an account to start building hosted sites
  • other CMS can be installed on your own web server
  • features and page customization vary greatly between CMS
  • Popular CMS include: Joomla, Drubpal, Movable Type, Modx, Radiant, WordPress, and Expression Engine

  Content delivery networks

  • CDN: a system for delivering content over a distributed network of servers
  • CDNs distribute content over many servers across many locations to speed up the serving of data.
  • Largely used to serve static resources such as JavaScript libraries, video, or other side dependencies.
  • leveraging CDN’s can reduce your server load and there is a chance that the user will have the library already cached due to visiting some other site that used the same CDN URL.
  • Types of CDNS:
  • Some CDN’s host a single type of resource.
  • CDN’s like YouTube serve not only as delivery networks, but content aggregators as well.
  • You can choose between numerous free or commercial CDNs for dedicated service.
  • Popular CDN’s include Akami, CloudFlare, Amazon’s CloudFront, EdgeCast and Windows Azure.
  • For dedicated resources, you don’t have the ability to create custom downloads or specific minified versions. (tree shaking is out on general purpose libraries).

  Cloud services

  • The World Wide Web - mostly just consists of websites, documents connected to other documents by hypertext links, enabling the user to search for information by moving from one document to another. (dictionary.com).
  • The Internet is a massive network of networks, a networking infrastructure. It connects millions of computers together globally, forming a network in which any computer can communicate with any other computer as long as they are both connected to the internet. (webopedia.com).
  • Cloud Computing
  • a distributed network of computers that can perform tasks by taking advantage of the scale and power of the network
  • usually refers to distributed process over the internet - As in not relying on a single server to solve a problem but multiple.
  • Services that take advantage of Cloud Computing typically make use of server-side languages such as Ruby, .NET, Java, JavaScript, Python etc.
  • The main difference is how they are architected and served.
  • The apps need to work on many servers and for a large number of users at the same time.
  • This has led to the rise of 3rd party services such as Google Cloud Storage, Amazon Web Services, Microsoft Azure.
  • Cloud Based technologies have changed the internet from merely a delivery mechanism to a development platform.

  What is Github?

  • Github is an online distribution service that allows users to store repositories online that can be version controlled and shared by others.
  • Built around GIt, an open source version control system. Git allows users to save changes to files, correct errors, branch off in new directions.
  • Offers both free and commercial plans
  • features an easy to use web interface
  • How Github Works
  • You create a new project to github, loading the most current files
  • GitHub then syncs the project to your local files.
  • When you make changes, these changes can be pushed to Github
  • GitHub will only store the changes to the file
  • GitHub is public by default if you are not using a paid plan.
  • GitHub’s shining feature is not it as an online platform to store your repositories but as a way to share and collaborate with others.

Assembly Technologies

  JavaScript libraries

  • A collection of JavaScript methods and functions that make it easier to perform targeted tasks.
  • JavaScript libraries extend the functionality of native JavaScript. THey range from extremely focused sets like Date.js to the broader task-oriented libraries like jQuery.
  • To make use of a JavaScript library you just need to link to it in the HTML
  • Using a JS library can help you complete your features quicker
  • Most JS libraries are open source and as a result are better tested than private libraries.
  • Cons of using libraries
  • Libraries increase the size of your code, and often add additional server requests.
  • You can become tied to a specific library syntax - Not knowing how to select an element w/o using jQuery etc.
  • Without knowing native JS you’re not always making an informed choice as to when a library is appropriate.

  Frameworks and boilerplates

  • Collection of prebuilt CSS, HTML, and JS files designed to speed development of sites.
  • Boilerplate
  • A set of templates built around a specific starting point or goal
  • Meant to provide a starting point for building sites or apps
  • Framework
  • Usually have a specific focus on building a site in a specific way.
  • Frameworks are generally opinionated - there is a correct way to leverage them to craft a website in the Framework approved way.
  • Minimal Frameworks
  • Typically have a singular goal or focus
  • Designed to have a small footprint and help out in specific areas
  • Rapid Prototyping
  • Frameworks are often used to build HTML prototypes
  • Using a framework can dramatically speed up prototyping development and iteration
  • in most cases framework code is not used for final production - built in Rails to get customer buy in then rewritten in something more performant.

  CSS preprocessors

  • Scripting language that extends the functionality of CSS and must be compiled into native CSS code before publishing.
  • Often easier and faster to write than pure CSS
  • Preprocessor benefits
  • They add additional functionality that CSS lacks - variables, mixins
  • Many browser inconsistencies are dealt with automatically when the CSS is processed.
  • Popular CSS processors
  • Sass -  Written in Ruby, two distinct syntaxes to choose from
  • Less - Written in JavaScript, syntax closely resembles native CSS
  • Stylus - Written in JavaScript, minimal syntax
  • Reasons against preprocessor
  • everyone needs to understand the preprocessor who might work with the site styles.
  • processed css can be bloated and inefficient (redundant style applications?).
  • Additional layer of complexity to workflows.
  • Some of the nice features of preprocessors are slowly making their way into native CSS. So in the future they may not be needed at all.
