So, since using GitHub Pages is new to me I’ve did a little reading. There Is a lot of useful information located on here. With further useful links at the bottom.

Some TLDR points about GitHub Pages:

  •   Soft 100gb bandwidth limit
  •   GitHub Pages source repos have a 1gb size limit
  •   GitHub Pages have a soft limit of 10 builds per hour*
  •   Any GitHub Pages site is accessible on the internet, even if it’s in a private repo

Project Pages

If you decide to create host some other projects on GitHub you can a GitHub page for that project as well, we aren’t limited to a single GitHub page.

*because of the build limit, there could be a delay when making changes to your site and having those changes applied to your GitHub Pages site.

For learning basic Git commands I think this video is short and to the point and covers many of the most common commands. Except for git log and git diff. Which are pretty important I, think.

git diff : a command to show changes between commits, commit and working tree, etc 

git log : show commit logs, very useful. If you want to checkout an older commit of you will need git log to find the hash associated with the commit.

git log --graph : graph where your branches diverged, where you head is at etc. Useful for visualizing your commit history.

git log --oneline  : show the same output as git log but make the output more terse. Easier to read most of the time

Pretty easy. Although I cloned my CIT-93 GitHub repository by habit. I was a little confused for a moment to find a .gitignore already present. Whoops! I would like to mention that if you don't like having to type out the commit message in the command line, it is common to use a simple text editor. You can set which editor to use, by running the command:

$ git config --global core.editor WhateverEditorYouDecideToUse

Also, just because you set a default editor doesn't mean you have to use it every single time, it should only launch when you neglect the message flat (-m) when you run your git commit.
