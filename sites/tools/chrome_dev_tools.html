Google Chrome

Developer Tools

Every good browser these days come with their own developer tools. Developer Tools allow Web Developers to
troubleshoot their code and diagnose bugs. Gaining expertise with these browser based tools
has become a necessity when developing for the web.

 

Google Developer Tools - Audit tab

This guy goes over a lot of the basics of Chrome's DevTools but I wanted to highlight his Audit portion of the video. I have never used the DevTools Audit tab and really didn't know much about it. I ended up running an audit on this webpage to see what results came up. If you wanted to run it yourself on this page, make sure you deselect the 'progressive web app' check box. Canvas already has a mobile app so I don't think it's fair to critique their App against Progressive web app best practices. Here are the results I got when I ran an Audit on 8/27/17 ( running an Audit on Hands-On webpage ). The largest hit to performance came surprisingly from a 65kb file. It wasn't the downloading of the file that caused the performance hit but probably applying all of the CSS styles. The total rendering delay was nearly 1 second, the other CSS files had substantial rendering delay times also but I'm not sure if those are actual time delays that the webpage experienced. My reasoning behind this is that if you scroll down on the Audit results, you should find a rendering/painting timeline and the output that gave me for the first meaningful paint (when a large portion of the page is not only loaded but rendered to our screen) was below 2 seconds. And if you add up all of the delays from the CSS alone, it amounts to 4 seconds. The audit results breaks down some suggestions to improve your score and I found that worth diving into. Especially when they link to outside resources such as Render-Blocking Resources.

 

30 Chrome DevTools Tips - Network Tab portion

Because the Audit tab did a performance analysis of the webpage loading, I was interested in what the Network Tab might show. This video is by nodecasts, in the video Jon goes over in some nice detail the breakdown that the Network tab does for you. My network tab will keep recording until i manually stop recording (the record button in the top left of the Network Tab). I used the network tab to check the results against what the audit tab was showing me. And I do see some discrepancies between the two. Based on the amount of information that the Network tab give you I would trust that, in terms of delay tracking over the Audit tab. With that said however, the Audit tab gives out recommendations while the network tab is all business. I think both will be very useful to me.
