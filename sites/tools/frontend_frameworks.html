Front-End Frameworks have become increasingly popular when developing for the web. Open source
JavaScript and CSS frameworks have allowed for greater ease in the creation
of complicated Web Apps and for quickly adding styling
and interactivity to sites.

Materialize CSS

Based on Google's Material Design spec, Materialize takes the Design language defined by Google and attempts to translate it to the web in the form of a front end framework. If you have any Android device that is running Android 5.0 and above, you have probably seen Apps that leverage Material Design.

Material Design is a push by Google to try to create a cohesive design standard for the Android environment. If you ever hear about a cross platform desktop application not looking as good on Windows or OSX its most likely because the Application was designed with on OS in mind and then ported to the other OS w/o taking into considerations the prevalent design language present on the other platform. When it comes to mobile, sometimes there are iOS apps, which are ported to Android but the app looks like an iOS app and feels out of place on Android. It is in every platform's best interest to have a cohesive design and to have 3rd party apps look and feel at home when they run in that environment. Similarly both Apple and Microsoft have started pushing their own design philosophies for their respective platforms, with Apple's message probably the least heavy handed.

Material Design

Materialize CSS is essentially using Google's Material Design spec and recreating the functionality for Web.

The strong points of the framework are:

  • opinionated, the framework tries to make some decisions for you
  • cohesive design language
  • fast to get started

Some of the disadvantages

  • opinionated, the framework makes decisions for you and limits choice
  • It can be easy for sites to look too similar and not stand out
  • Its very large, one of the largest CSS libraries
  • It's not mature, when compared to other CSS Frameworks.
