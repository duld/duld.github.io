Gulp JS

JavaScript Taskrunner

Taskrunners are very popular for automating repetitive tasks when doing Web Development. Often these tools
are able to free up time for the developer and move repetive
menial tasks to the background.

Gulp
Gulp - Github

"gulp is a toolkit for automating painful or time-consuming tasks in your development workflow, so you can stop messing around and build something."

Gulp Breakdown

The best breakdown of "what is gulp" I found

Essentially Gulp along with other task runners will add some automation to your build process. By build process I mean, whatever steps you take to make your development code (the code you write when adding features) into deployment code (the code that the user of your app/site receives). In the context of this week's Monday Current Topics - Optimizing Images examples, we could set up a task to automatically compress any new images we add to the site using a package such as imagemin.

Minification

Another common task is concatenating .css and .js files and then minifying those files. This can reduce the number of requests required to load your website, which could cause it to load much quicker.

Cons

Some cons to working with task runners. Based on what I've read, task runners often rely on plug-ins that can have little to no documentation, or they are out of date and you must wait for the plug-in to update. Often these plug-ins will leverage the API's of an NPM package but will offer a different public API to the developer using Gulp/Grunt. This can cause confusion and mixed output based on the NPM package API docs and the Gulp plug-in API docs.

In conclusion..

I personally don't know enough about task runners to form any kind of conclusion, but I have heard people mention Gulp, Webpack and Yeoman offering desirable features. From all the advice I've seen so far, everyone seems to agree that you should start with just Node and NPM and leverage those tools until you need something more. Although some argue that you can just stick with Node and NPM.
