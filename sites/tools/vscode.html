Visual Studio Code

Microsoft's Visual Studio Code is a free and powerful code editor. It lies somewhere
between Sublime Text and Visual Studio IDE. VSCode has quickly become one
of the most widely used editors, Good Job MS Devs!

Visual Studio Code for Web Developers (lynda.com) - main editor

Overview of VS Code

The VS Code UI

  • launch VS Code
  • powershell, git bash, cmd: code <directory>*  optional
  • Viewbar: The vertical list of icons on the left side of the screen
  • Explorer view (ctrl + shift + e) - shows all files and directories within the open project folder along with what files have unsaved changes.
  • Search view (ctrl + shift + s) - search for content across multiple files
  • Integrated Git View (ctrl + shift + g) -  The integrated git view allows you to see what uncommitted changes have been made
  • Debug View (ctrl + shift + d)
  • Extension View (ctrl + shift + x) - Add, Remove or Update extensions
  • Command Palette (ctrl + shift + p) - access all the features of VS code from a scrollable list.
  • Zen mode: different view mode to only show the current working file.

Basic editing

  • copy, cut, paste line of code: with the cursor on the line in question perform the copy, cut or paste to perform the operation on the entire line.
  • Indent / Dedent: ctrl + >, ctrl + <
  • Format document: alt + shift + f
  • Format Selection: ctrl + k then ctrl + f
  • Increase Selection: ctrl + i *works top down*
  • Move lines up and down: alt + up arrow, alt + down arrow
  • Vs code will always try to keep braces balanced as best it can automatically

The integrated terminal

  • launch: ctrl + ~
  • you can have multiple terminal
  • you can run highlighted text from your editor in your terminal by:
  • typing out the command you would like
  • highlighting the command: ctrl + i
  • launch the command palette: ctrl + shift + p
  • type “run selected text in active terminal” BLAM!
  • move up or down inside terminal: ctrl + arrow up/down
  • clear terminal: ctrl + k

Integrated Source Control

  • leverages the system’s default installation of git. requires git 2.0 minimum
  • uncommitted changes will be shown in the Integrated Git View
  • If you select one of the files with unsaved changes you can get a document style diff view, where VS Code will clearly show you any insertions or deletions.
  • Gutter Annotations
  • Green bar: New line added
  • Blue bar: Current line is changed
  • Red triable: Line has been deleted
  • Staging / Unstaging files: click the “plus icon” next to a file. multiple files can be staged at once. To unstage a file, click the ‘minus icon” next to the file.
  • To see a history of the git commands that were used: click the More icon, select Show Git output

Customizing VS Code

Configuring preferences

  • file -> preferences or ctrl + , (comma)
  • settings are stored in a settings.json file
  • the left side of the screen is read only, the left side is edible.
  • create language specific settings: open the command palette (ctrl + shift + p) and type “Preferences: Configure Language Specific Settings”

Setting keyboard shortcuts

  • File->Preferences->Keyboard Shortcuts
  • keybindings have three significant attributes
  • key: what is the key or key combination to map a command to
  • command: what is the command you will be running
  • when: in what context will this key binding be able to call the command

Using extensions

  • marketplace.visualstudio.com/vscode
  • extensions view (ctrl + shift + x)
  • install an extension: search for an extension, click install.
  • Uninstall an extension: click the gear icon and, turn off the extension or uninstall

Essential extensions

  • Language snippets: language snippets can be very useful, an HTML language snippet for example can speed up your workflow incrementally by performing some auto indenting and cursor placement.
  • Beautify: format your document based on editable preferences to increase readability
  • Express: use the express extension to have a built in development web server.
  • ctrl + shift + p - Express: hostWorkspaceWithRandomPortAndOpenInBrowser
  • to launch your project
  • REST Client: test REST APIs by writing REST commands in a .http file. You can run the command in the file and view a textual output.

Working with Snippets

  • snippet: small blocks of code that can be inserted
  • define user snippet:
  • file->preferences->user snippet
  • select language
  • snippets are defined using JSON.
  • snippet anatomy:
  • <snippet name> : {
    “prefix” : <text user will enter to prompt snippet>,
    “body” : [<string of code to be inserted>],
    “description” : <description of snippet to show user>
    }
  • body cont. : in the body portion of the snippet you can define tab-stops and predefined values for the snippet to be inserted. to define a tab stop you use the following syntax. ${1:placeholder}
  • “1” in this instance designates that this is the first tab stop, the next tab stop will have a value of 2, then 3 and so on.

Color and icon themes

  • VS Code comes with a set of default color themes
  • you can find additional color themes on the marketplace.
  • icon themes: you can turn on icon themes from the command palette

Using the Editor

Files, folders, and projects

  • hide/show sidebar: ctrl + b
  • preview mode: select a file and a preview of the file will be displayed. you can select a different file and the contents of that file will be displayed. The tab is reused, no additional tabs will be opened to manage.
  • edit mode:  double clicking the file or making a change to a file in preview mode will open that file in edit mode. A tab in this mode is now dedicated to the file.
  • Cycle through Tabs: ctrl + tab
  • Close other editors: ctrl + shift + p, close other editors
  • note: in the video, ctrl + alt + t is said to perform this action but the that specific command is not mapped to a shortcut combination in my VS Code.
  • This may be due to that specific combination commonly being used in Linux distributions to launch the Terminal. I know this is the case for Ubuntu and Linux Mint.
  • Reopen recently closed tab: ctrl + shift + t
  • Display file/folder in File Explorer: right click file->Reveal in Explorer
  • open folder in terminal: right click file->open in terminal

Side by side editing

  • Open a file in side by side: right click a file->Open to the Side or ctrl + enter
  • You can have up to three split views: left, center, right

IntelliSense

  • VS Code comes with IntelliSense by default
  • Intellisense supports many languages out of the box and additional language support can be added through the marketplace.
  • activate IntelliSense: ctrl + space
  • IntelliSense can offer syntax suggestions or mini documentation to help when writing code. Often multiple suggestions can be present. Press the up or down keys to view other suggestions
  • huge topic, single video won’t cover it.
  • https://code.visualstudio.com/docs/editor/intellisense
  • https://code.visualstudio.com/docs/introvideos/intellisense

Finding and replacing

  • Find: ctrl + f
  • basic find in file
  • Find Replace: ctrl + h
  • find by case, replace find results with replace value
  • Find in File: ctrl + shift + f
  • find search string in multiple files
  • Increased Find settings: ctrl + shift + j
  • find and replace across multiple files, exclude files by extension

Advanced Editing

Editor features

  • Hot Exit: VS Code will store changes temporarily if you close the editor on accident.
  • Auto format on save: you can enable file formatting when you save the file, but this can cause problems if you are opening someone else’s file and formatting their file. If there is an agreed upon format styling then it will not be an issue however.

Web code editing

  • Toggle comments: ctrl + /
  • JSDOC comments: /**  will initialize a JSDOC block
  • Markdown: syntax highlighting and live preview of markdown files.
  • HTML live preview: this feature seems to be removed from the current version of VS Code on Windows 10.
  • Emmet:
  • docs.emmet.io 
  • docs.emmet.io/cheat-sheat/
  • quickly write some html tags using simplified commands
  • ! - quickly writes html doc boilerplate
  • ul>li*6 - creates an unordered list containing 6 elements.

Navigating code

  • Go to specific line: ctrl + g
  • go to specific symbol: ctrl + shift + o
  • open recently closed files: ctrl + p

Using multiple selections

  • select word: ctrl + d
  • select all occurrences: ctrl + shift + l
  • insert multiple cursors: ctrl + alt + up/down arrow
  • note: this may rotate your windows screen if you have intel graphics. You should disable your intel graphics hotkey settings to gain access to this feature.
  • cheat sheat - https://code.visualstudio.com/shortcuts/keyboard-shortcuts-windows.pdf
  • increase selection: ctrl + shift + right / left arrow

JSON and schemas

  • VS Code has tooltips for common JSON schemas used for different libraries.
  • schemastore.org
  • Add schema to workspace
  • go to settings: ctrl + ,
  • search for json.schemas
  • create a json entry that follows the VS Code syntax for defining a schema to be used.
  • https://code.visualstudio.com/docs/languages/json#_json-schemas-settings
  • too much referenced in example. Just follow the docs lol.

Learning Atom (lynda.com)  - side editor

Using the Atom Interface

Managing and folding content

  • Fold/Unfold content: ctrl + alt + [, ctrl + alt + ]
  • Fold Unfold all by level: ctrl + k followed by ctrl + 1-9 for level of folding
  • note: it appears as if folding relies on indentation and whitespace to function sometimes, my <div> container for example would not fold properly until I indented the content contained within the <div>

Working with Atom and Git

  • Bottom bar Git GUI
  • displays which branch you are in
  • shows number of line deletions / insertions
  • Gutter
  • Orange Line: unsaved changes
  • Red arrow: removed line
  • Green Line: new line
  • open list of open files: ctrl + b: files with uncommitted changes will show a square icon with a dot in the center.
  • show all files with uncommitted changes: ctrl + shift + b
  • discard current uncommitted changes: ctrl + alt + z
  • get back discarded changes: ctrl + z

Atom’s GitHub Integration features

  • open file on github: ctrl + g o
  • open the blame file on github: ctrl + g b
  • history of file: ctrl + g h
  • issues: ctrl + g i
  • copy github file url: ctrl + g c
