Why to avoid JavaScript's 'with' statment

I noticed while reading the BNR book that all of the JavaScript functions have the 'use strict' string statement placed at the top of the definition. When I looked into, 'use strict', I began to see the reasons why this was the case. One such reason was to disallow the use of the JavaScript 'with' statement.

The 'with' statement in JavaScript is now depricated and it's use if frowned upon. It has been attributed to confusing behavior and going forward into ES6 will not provide the same functionality as it did in ES5. The main causal of the confusion regarding 'with', stems with how the statment handles scope.

The with statement makes it hard for a human reader or JavaScript compiler to decide whether an unqualified name will be found along the scope chain, and if so, in which object.-expert from MDN
